PARTIE 1: BOT DISCORD (discord.py)
──────────────────────────────────

Fichier: discord.py

Classe: Discord Bot (async)
├─ Événements:
│  └─ on_ready() → Affiche confirmation de connexion
│
└─ Commandes:
   ├─ @exporter_json
   │  ├─ Récupère tous les rôles du serveur
   │  ├─ Filtre @everyone
   │  ├─ Extrait: id, name, position, color, permissions_value
   │  ├─ Génère JSON avec indentation (lisible)
   │  ├─ Crée fichier virtuel (BytesIO)
   │  └─ Envoie dans le salon
   │
   └─ @importer_json
      ├─ Lit le fichier JSON attaché
      ├─ Parse la structure
      ├─ Crée rôles (is_new: true)
      │  ├─ discord.create_role(name, color, permissions)
      │  ├─ Gère les erreurs
      │  └─ Confirme chaque création
      ├─ Bulk Edit positions
      │  └─ guild.edit_role_positions({role: position})
      └─ Envoie rapport avec embed

Intents activés:
├─ message_content → Lire contenu des messages
├─ guilds → Événements serveurs
└─ guild_roles → Événements rôles

Sécurité:
├─ Token stocké en .env (via python-dotenv)
├─ Aucun rôle supprimé (préservation)
├─ Bulk edit en 1 requête (pas de rate-limit)
└─ Logs console pour debugging

═══════════════════════════════════════════════════════════════════════════
 📦 FORMAT JSON
═══════════════════════════════════════════════════════════════════════════

Structure d'Export:
{
    "id": 1104570726405320733,           // ID Discord unique
    "name": "Admin",                     // Nom du rôle
    "position": 10,                      // Position hiérarchie (0 = bas)
    "color": [88, 101, 242],            // RGB (0-255 chacun)
    "hex_color": "#5865f2",             // Format Hex (#RRGGBB)
    "permissions_value": 8,              // Bitfield Discord (entier 64-bit)
    "is_new": false                     // true = à créer, false = existant
}

Flux JSON:
1. Export: Bot génère avec is_new: false
2. Web UI: Modifie, crée rôles avec is_new: true
3. Import: Bot lit is_new, crée si nécessaire
4. Final: Tous les rôles avec positions recalculées

Scénario: Créer 5 nouveaux rôles + réorganiser existants

Étape 1: Export (Bot Discord)
─────────────────────────────
User: !exporter_json
Bot:
  ├─ Récupère 50 rôles (ignore @everyone)
  ├─ Crée structure JSON avec is_new: false
  ├─ Envoie roles_configuration.json
User:
  └─ Télécharge le fichier

Étape 2: Modification (Interface Web)
────────────────────────────────────
User:
  ├─ Ouvre index.html
  ├─ Importe roles_configuration.json
  ├─ Web UI affiche 50 rôles
  ├─ Crée 5 nouveaux (modal):
  │  ├─ Modérateur (permissions: KICK, BAN)
  │  ├─ Helper (permissions: MANAGE_MESSAGES)
  │  ├─ etc.
  │  └─ Chacun: is_new: true
  ├─ Réorganise positions (haut/bas)
  ├─ Exporte: roles_configuration_modified.json
  └─ Télécharge fichier

Étape 3: Import (Bot Discord)
────────────────────────────
User: !importer_json [+ fichier]
Bot:
  ├─ Parse le JSON
  ├─ Crée 5 nouveaux rôles (is_new: true)
  │  ├─ discord.create_role("Modérateur", permissions=..., color=...)
  │  ├─ discord.create_role("Helper", ...)
  │  └─ Chacun confirmé
  ├─ Bulk Edit: réorganise 55 rôles en 1 requête
  ├─ Envoie rapport:
  │  ├─ ✅ 5 rôles créés
  │  ├─ ✅ 55 positions réorganisées
  │  └─ ✅ Succès!
  └─ Discord Server: MIS À JOUR ✅


Étape 4: Vérification
───────────────────
User:
  ├─ Regarde le serveur Discord
  ├─ 5 nouveaux rôles créés ✅
  ├─ Hiérarchie réorganisée ✅
  ├─ Aucun rôle supprimé ✅
  └─ Permissions correctes ✅
